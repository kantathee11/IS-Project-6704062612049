# เอกสารประกอบโครงการ (Project Documentation)

**ชื่อโครงการ:** ระบบพยากรณ์ความเสี่ยงสุขภาพและวิเคราะห์ความรู้สึกด้วย AI  
**รหัสนักศึกษา:** 6704062612049

---

## 1. แนวทางการพัฒนาโมเดล (Model Development Approach)

### การเตรียมข้อมูล (Data Preparation)
* **ที่มาของข้อมูล:** โปรเจกต์นี้ใช้ข้อมูลที่ผ่านการสังเคราะห์ให้มีความใกล้เคียงกับสถานการณ์จริง โดยแบ่งเป็น 2 ชุดข้อมูลหลัก ได้แก่ `heart_data.csv` (ข้อมูลสุขภาพพนักงาน) และ `review_data.csv` (ข้อมูลรีวิวภาษาอังกฤษ)
* **การจัดการข้อมูล:** มีการตรวจสอบความถูกต้องและทำความสะอาดข้อมูล (Data Cleansing) โดยการลบข้อมูลซ้ำซ้อน (Drop Duplicates) และจัดการกับค่าว่าง (Dropna) เพื่อให้ Dataset มีคุณภาพสูงสุดก่อนนำไปสอนโมเดล
* **การสร้างสถานการณ์:** จำลองข้อมูลให้มีความหลากหลาย ทั้งช่วงอายุ (Age), ระดับคอเลสเตอรอล (Cholesterol) และระดับความเครียด (Stress Level) เพื่อให้โมเดล Machine Learning สามารถเรียนรู้รูปแบบความเสี่ยงสุขภาพได้ครอบคลุม

### ทฤษฎีของอัลกอริทึม (Algorithm Theory)
โปรเจกต์นี้เลือกใช้เทคนิคการประมวลผลที่ทันสมัย 2 รูปแบบหลัก:

**1. Ensemble Learning (สำหรับการพยากรณ์สุขภาพ):**
ใช้เทคนิค **Voting Classifier** เพื่อรวมพลังของ 3 โมเดลที่แตกต่างกัน เพื่อลดความลำเอียง (Bias) และเพิ่มความเสถียร:
* **Random Forest:** ใช้หลักการ Bagging สร้างต้นไม้ตัดสินใจหลายต้นเพื่อลดความแปรปรวน (Variance)
* **Logistic Regression:** ใช้การหาความสัมพันธ์เชิงเส้นเพื่อระบุโอกาสเกิดความเสี่ยงแบบเป็นเหตุเป็นผล
* **Support Vector Machine (SVC):** ใช้การสร้างขอบเขต (Margin) ที่กว้างที่สุดเพื่อแยกกลุ่มข้อมูลสุขภาพ

**2. Neural Network (สำหรับการวิเคราะห์ความรู้สึก):**
ใช้โครงสร้าง **Multi-layer Perceptron (MLP)** สำหรับงาน Sentiment Analysis:
* **MLP Classifier:** จำลองการทำงานของระบบประสาทมนุษย์ มี Hidden Layers ในการเรียนรู้ความสัมพันธ์ที่ซับซ้อนของคำศัพท์ภาษาอังกฤษ
* **TF-IDF Vectorizer:** แปลงข้อความ (Text) ให้เป็นตัวเลขเชิงสถิติ เพื่อให้ Neural Network สามารถประมวลผลความหมายของคำชมหรือคำตำหนิได้

## 2. ขั้นตอนการพัฒนาโมเดล (Development Process)

* **Data Cleaning & Integration:** นำ Dataset ทั้งสองส่วนเข้าสู่กระบวนการ Preprocessing ลบอักขระพิเศษในรีวิว และปรับสเกลตัวเลขในข้อมูลสุขภาพให้พร้อมใช้งาน
* **Feature Engineering:** ในส่วนของ Neural Network มีการแปลงข้อความภาษาอังกฤษเป็นค่าน้ำหนัก TF-IDF และในส่วนของ Machine Learning มีการนำตัวแปร Age, Cholesterol และ Stress มาใช้เป็น Feature หลักในการทำนาย Target
* **Data Splitting:** แบ่งข้อมูลเพื่อใช้สำหรับการ Training และการประเมินผล เพื่อตรวจสอบความแม่นยำ (Accuracy) ของแต่ละโมเดล
* **Model Training & Ensemble:** ฝึกสอนโมเดลทั้ง 3 ประเภทในส่วนของ Health และฝึกสอน Neural Network ในส่วนของ Sentiment พร้อมกันบนระบบ Streamlit
* **Model Export:** บันทึกโมเดลที่ฝึกเสร็จแล้วลงในไฟล์ `.pkl` (เช่น `ensemble_model.pkl`) ภายในโฟลเดอร์ `models/` เพื่อให้แอปพลิเคชันสามารถโหลดมาใช้งานได้ทันที (Re-usable)
